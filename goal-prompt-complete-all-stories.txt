## Quy trình cho MỖI story (thực hiện đúng thứ tự)

1. Đọc story packet tại `docs/stories/epics/E**/US-0**.md` (và plan nếu có).
2. Tạo branch từ main mới nhất:
   ```bash
   git checkout main && git pull
   git checkout -b feat/us-NNN-<slug>
   ```
3. Implement theo đúng Acceptance Criteria + Design Notes trong packet.
4. Chạy validation từ repo root:
   - `pnpm -F @ziweiai/web check` (nếu chạm web)
   - `pnpm -F @ziweiai/api test` hoặc `turbo typecheck`
   - `pnpm lint`
   - Test liên quan phải xanh
5. Gọi code reviewer agent → fix hết CRITICAL + HIGH findings.
6. Cập nhật harness:
   ```bash
   scripts/bin/harness-cli.exe story update --id US-NNN --unit X --integration Y --e2e Z --platform W
   scripts/bin/harness-cli.exe trace ...
   ```
7. Commit + push + tạo PR (title ngắn, body có summary + test plan + link story packet).
8. Đợi người dùng review. Chỉ khi người dùng nói "done" cho PR này thì mới chuyển sang story tiếp theo.

## Quy tắc chung
- Mỗi story đúng 1 PR, không gộp.
- Ưu tiên Wave 1 → Wave 2 → Wave 3.
- Wave 1 có thể làm tuần tự 012 → 013 → 017 → 018.
- Sau khi US-014 merge vào main mới được chạm US-015 và US-016.
- Luôn chạy lệnh từ thư mục gốc repo.
- Tôn trọng CLAUDE.md + rules (Han guard, immutable, boundary...).
- Gặp blocker → dùng `scripts/bin/harness-cli.exe backlog add`.
- Cuối cùng chạy `scripts/bin/harness-cli.exe query matrix` để xác nhận.
